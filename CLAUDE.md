# CLAUDE.md - wren-mcp

See @README.md for what this project is.

Local **stdio** MCP server that gives an AI assistant read access to a Wren notes
folder. It is a read-only **consumer** of Wren's AI-readable layer (stable
`wren-...` frontmatter ids, the frozen `.wren-index.json` catalog, and the
`_inbox/` staging subfolder) plus one write path that creates notes into
`_inbox/` only. TypeScript/ESM, packaged as a `.mcpb` Desktop Extension.

## Build, test
- `npm run build`     # tsc -> dist/
- `npm run dev`       # tsc --watch
- `npm test`          # vitest run (must pass before any commit)
- `npm run lint`      # eslint . (must pass before any commit)
- `npm start`         # node dist/index.js (run the built server over stdio)
- `npm run inspect`   # build, then launch the MCP Inspector against the server
- `npm run pack`      # build + prune dev deps + mcpb pack -> Wren.mcpb, then restore
- CI (.github/workflows/ci.yml) runs lint + build + test on every PR; the
  required status check is `build`.

## Branching (main is protected)
`main` is protected - direct pushes are rejected. Branch, commit, push, open a
PR, then squash-merge once CI is green. Never run `git push origin main`.

## File organization (root is locked)
Do not add files to the repo root unless required. Before creating any new file:
1) identify which folder it belongs in, 2) create it if missing, 3) add it there.
- New source -> `src/`; new tests -> `tests/`; docs -> `docs/` (internal
  planning -> `docs/internal/`).
- `manifest.json` AND `icon.png` legitimately stay at the root: the `.mcpb`
  Desktop Extension package format requires both at the bundle root. This is a
  hard requirement of the format, not a root-locking violation.

## Do not touch
- `.env` / `.env.local` hold local secrets - never commit or print them. Commit
  `.env.example` only (`.gitignore` already ignores `.env` / `.env.*` except the
  example).
- `dist/` is generated by `tsc` - never edit by hand.
- `Wren.mcpb` is a generated bundle (`npm run pack`) - never edit by hand; it is
  gitignored.
- The `.wren-index.json` schema (schemaVersion 1) is a **frozen cross-repo
  contract** owned by the Wren PWA, not this repo. This server only reads it - do
  not casually change the shape it parses or the fields it relies on; a change
  here must track a deliberate schema bump on the Wren side.
